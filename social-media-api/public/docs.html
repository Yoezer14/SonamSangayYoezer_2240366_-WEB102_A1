Social Media API

REST endpoints for users, posts, comments, likes, and followers (mock data). Base URL: http://localhost:3000

Content negotiation

Default responses are JSON (application/json). Send Accept: application/xml or Accept: text/xml for XML responses.

Write operations expect Content-Type: application/json.

Users

MethodURIDescription
GET/usersList users. Query: page, limit
GET/users/:idGet one user
POST/usersCreate user. Body: username, email, password, full_name?, bio?
PUT/users/:idUpdate user
DELETE/users/:idDelete user

Posts

MethodURIDescription
GET/postsList posts. Query: page, limit, user_id
GET/posts/:idGet one post
POST/postsCreate post. Body: user_id, caption?, image_url?
PUT/posts/:idUpdate post
DELETE/posts/:idDelete post

Comments

MethodURIDescription
GET/commentsList comments. Query: page, limit, post_id
GET/comments/:idGet one comment
POST/commentsCreate comment. Body: post_id, user_id, text
PUT/comments/:idUpdate comment
DELETE/comments/:idDelete comment

Likes

MethodURIDescription
GET/likesList likes. Query: post_id, user_id, page, limit
GET/likes/:idGet one like
POST/likesCreate like. Body: post_id, user_id
PUT/likes/:idUpdate like
DELETE/likes/:idDelete like

Followers

Represents “user A follows user B” as follower_id → following_id.

MethodURIDescription
GET/followersList relationships. Query: follower_id, following_id, page, limit
GET/followers/:idGet one relationship
POST/followersCreate follow. Body: follower_id, following_id
PUT/followers/:idUpdate relationship
DELETE/followers/:idDelete (unfollow)

Response shape (JSON)

  • Success lists: { success, count, page, total_pages, data }
  • Success item: { success, data }
  • Errors: { success: false, error, statusCode }
